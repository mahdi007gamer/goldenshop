# Admin Orders Pro — برنامه پیاده‌سازی

**تاریخ شروع:** 2026-06-30
**وضعیت:** در حال اجرا
**مسیر برنامه:** `docs/superpowers/plans/2026-06-30-admin-orders-pro.md`

## خلاصه

سیستم کامل مدیریت سفارشات شامل:
- **ادمین:** لیست سفارشات با فیلتر، تایید/رد پرداخت کارت به کارت، تخصیص اتوماتیک لایسنس از انبار، ارسال دستی لایسنس وقتی انبار خالیه
- **کاربر:** صفحه وضعیت سفارش با شمارنده ۶۰ دقیقه‌ای، پولینگ لحظه‌ای، انیمیشن‌های جذاب برای هر وضعیت (تایید شد، رد شد، لایسنس داده شد)
- **امنیت:** جداسازی کاربران، ریت لیمیت، ورودی‌های پاک‌سازی شده

## معماری

### بک‌اند
- **مدل‌های جدید:** `LicenseDelivery` (ردیابی ارسال لایسنس)، `OrderStatusLog` (لاگ تغییرات وضعیت)
- **APIهای جدید:**
  - `POST /api/admin/orders/[id]/deliver` — ارسال دستی لایسنس
  - `GET /api/admin/orders/stats` — آمار سفارشات
  - `GET /api/orders/[id]/status` — پولینگ وضعیت کاربر
  - `PATCH /api/admin/orders/[id]` — ذخیره یادداشت داخلی

### فرانت‌اند
- **صفحه ادمین:** `/admin/orders` — لیست، فیلتر، تایید پرداخت، ارسال لایسنس
- **صفحه کاربر:** `/checkout/[id]` — انیمیشن، شمارنده، نمایش لایسنس

### امنیت
- ادمین روت‌ها: `requireAdmin` از `@/lib/auth-utils`
- کاربر روت‌ها: `requireAuth` + بررسی مالکیت سفارش
- ورودی‌ها: `sanitizeString` روی همه ورودی‌ها
- لایسنس‌ها: رمزنگاری شده در دیتابیس
- ریت لیمیت: ۶۰ درخواست در ۱۰ ثانیه برای پولینگ

## قوانین مهم
- سفارشاتی که ۴ رقم آخر کارت وارد کردن باید مثل بقیه بررسی بشن
- ادمین اول پرداخت رو تایید می‌کنه، بعد لایسنس تخصیص داده میشه
- اگه لایسنس توی انبار نبود، ادمین دستی وارد می‌کنه
- صفحه کاربر ۶۰ دقیقه شمارنده داره، بعد تغییر وضعیت ریست میشه
